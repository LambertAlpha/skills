# Research OS 周报生成器

每周模型运行后，自动拉取三大模型输出，生成机构级宏观研究周报，发送到飞书 LAC 工作群。

## Script Location

脚本位于本 Skill 目录下的 `scripts/`：
```bash
SKILL_DIR="$(dirname "$(find ~/.claude/skills -name SKILL.md -path '*/weekly-report/*' 2>/dev/null | head -1)")"
```

## Before You Start

| 配置项 | 说明 | 设置方式 |
|--------|------|---------|
| **飞书 Webhook** | LAC 工作群的机器人 Webhook URL | `echo "https://open.larksuite.com/open-apis/bot/v2/hook/xxx" > ~/.lark_webhook && chmod 600 ~/.lark_webhook` |
| **Research OS API** | 后端 API 地址 | 默认 `http://202.81.229.139:8000`，可通过 `--api-url` 覆盖 |

## Quick Start

```bash
# 1. 拉取最新模型数据
python3 $SKILL_DIR/scripts/fetch_model_data.py

# 2. [Claude 根据下方风格指南生成周报]

# 3. 发送到飞书
python3 $SKILL_DIR/scripts/send_to_lark.py --webhook @~/.lark_webhook --markdown --title "Research OS 周报 YYYY-MM-DD" "报告内容"
```

## When This Skill Activates

- 用户说 "生成周报"、"写周报"、"发周报"
- 用户说 "run weekly report"、"/weekly-report"
- 每周五模型运行后（可配合 hook 自动触发）

## Workflow

**严格按以下步骤执行：**

### Step 1: 拉取数据

```bash
python3 $SKILL_DIR/scripts/fetch_model_data.py --api-url http://202.81.229.139:8000
```

如果连不上远程服务器，尝试本地：
```bash
python3 $SKILL_DIR/scripts/fetch_model_data.py --api-url http://localhost:8000
```

### Step 2: 生成周报

**用你（Claude）的能力**，根据拉取到的数据 + 下方的写作风格指南，生成一份完整周报。

**不是简单翻译模型输出，而是写一份有观点、有逻辑链条、有可操作建议的研究报告。**

### Step 3: 展示给用户确认

将生成的周报展示给用户，等待确认后再发送。

### Step 4: 发送到飞书

用户确认后：
```bash
python3 $SKILL_DIR/scripts/send_to_lark.py --webhook @~/.lark_webhook --markdown --title "Research OS 周报 YYYY-MM-DD" "REPORT_CONTENT"
```

如果报告太长，用 `--stdin`：
```bash
echo "REPORT_CONTENT" | python3 $SKILL_DIR/scripts/send_to_lark.py --webhook @~/.lark_webhook --markdown --stdin
```

## 写作风格指南（核心）

### 总体原则

你是 Agarwood Technology 的首席宏观策略師。你的读者是公司内部研究员和交易团队。

**核心要求：写分析，不是列数据。** 数据在 Dashboard 上随时能查到，周报的价值在于**解读**——这些数字背后反映了什么，边际变化意味着什么，我们该怎么应对。

1. **观点驱动**：先给结论，再用数据佐证。不是"MOVE=105"，而是"债市波动率回升反映市场对 CPI 的避险情绪"
2. **讲故事**：把三个模型的信号串成一条叙事线——流动性环境如何影响宏观判断，宏观状态又如何映射到美股配置
3. **聚焦边际变化**：读者不需要知道每个数字是多少，只需要知道"什么变了，为什么重要"
4. **行动导向**：每段分析都要回答"所以呢？该怎么做？"

### 语言风格

- 使用**繁体中文**（團隊慣用語言）
- 金融术语保持英文原文：MOVE、SOFR、OAS、Steepener、Real Yield
- 语气专业直接，给出明确判断——不用"可能"、"或许"、"我认为"
- 数据只在支撑论点时引用，不单独罗列

### 报告结构与排版规范

**排版核心原则：**
- 每个大段落之间用 **emoji 标题 + 分隔线** 隔开，视觉上一目了然
- **关键结论加粗**，让快速扫读的人 3 秒抓住重点
- 用 `>` 引用框做「重点摘要」和「行动要点」的高亮区块
- 段落保持短小（3-5 行），避免大段文字墙
- 数字出现时用 **加粗** 强调关键值（如 **87.5/100**、**🟢 Green**）

```markdown
# 📊 Agarwood 宏觀週報（MM/DD）

> **本週一句話：**流動性環境維持寬鬆但信用閘門亮紅燈，宏觀敘事正從「增長」向「折現率衝擊」切換，建議降低方向性敞口、增加結構性對沖。
```

### 排版速查表

| 元素 | 用法 | 效果 |
|------|------|------|
| `> 📌 **關鍵判斷**：...` | 核心结论高亮框 | 引用框 + emoji + 加粗 = 视觉焦点 |
| `> 💡 **策略含義**：...` | 每段末的行动要点 | 快速找到"所以呢" |
| `> 📊 **體制：XXX**` | 关键状态数据块 | 一行展示核心参数 |
| `**关键数字**` | 行文中的重要数值 | 扫读时抓住关键 |
| `---` | 大章节分隔线 | 视觉呼吸空间 |
| `## 🎯 / 💧 / 🌍 / 📈 / 🔭` | 章节标题 emoji | 一秒定位章节 |

### 写作标准：正确 vs 错误

**❌ 数据陈列（不要这样写）：**
> **流動性評分**：87.5/100
> **風險燈號**：🟢 Green
> **槓桿係數**：1.0x
> **SOFR-IORB**：-3.0bp（正常）
> **MOVE**：84.9（暢通）
> **Reserves 4W 趨勢**：上行

**✅ 分析解读（要这样写）：**
> 美元流動性環境本週維持寬鬆格局。聯準會資產負債表縮表節奏未變，但銀行準備金 4 週趨勢持續上行，顯示系統中的流動性緩衝仍然充裕。貨幣市場壓力指標（SOFR-IORB）處於負值區間，遠低於 3bp 的關注線，資金面沒有任何緊張跡象。MOVE 指數 84.9 遠低於 100 的關注閾值，利率市場波動率處於低位。
>
> 簡言之，流動性三個維度（量、管道、波動率）全部亮綠燈，這是自 X 月以來最寬鬆的配置——槓桿可以正常使用，賣波動策略暢通。

**❌ 分层罗列（不要这样写）：**
> **Layer 1**：政策路径 Front-end Up，曲线 Bear Flatten，2s10s=49.9bp...
> **Layer 2**：Corr 20D=-0.16，60D=-0.06，叙事状态 transition...
> **Layer 3**：MOVE 🟢，SOFR-IORB 🟢，信用 🔴...

**✅ 综合叙事（要这样写）：**
> 利率市場正在釋放一個矛盾信號：短端在 price in 更鷹派的 Fed（2Y 上行），但長端並未跟隨（曲線熊平），這通常意味著市場擔心過度緊縮會拖累經濟。股債相關性接近零（Corr20D=-0.16），敘事處於過渡期——市場既沒有堅定地交易「增長」，也沒有完全切入「衰退」模式。
>
> 值得注意的是，信用閘門本週亮了紅燈——HY-IG 加速走闊。這是一個需要認真對待的信號：當高收益債利差開始加速惡化時，往往是風險偏好轉向的早期警告。結合曲線熊平和敘事過渡的背景，我們傾向將宏觀狀態判定為 C（衰退/風險厭惡），建議以防守為主。

### 告警处理

如果模型输出包含 CRITICAL 级别告警，在报告**最顶部**增加警告：

如果某些数据无法获取（API 故障、指标 pending 等），不要写 "N/A" 或留空白——直接跳过那部分，在报告中自然地注明数据限制。
